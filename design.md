## 1. 架构设计（行业新闻简报）

目标：每天自动从多个内容平台采集行业相关信息，进行去重、质量评估与热点聚合，输出结构化新闻简报，并通过用户反馈持续调整agent和skills的抓取结果。

1.2 模块 / Agent 职责划分
OrchestratorAgent：制定执行计划（按平台并行采集 → 清洗 → 去重聚类 → 质量评分 → 洞察 → 生成简报），负责超时、重试、失败降级与进度汇报。

Collector Agents（按平台拆分）：对接数据分析 API/爬取方式，产出原始条目（content + metadata + engagement + source_url）。

Normalize&EnrichAgent：统一数据模型（跨平台字段对齐），做文本清洗、语言检测/翻译、实体/公司/产品/人物抽取、链接展开、发布时间对齐与时区归一。

Dedup&ClusterAgent：
  - 去重：同 URL、同视频 ID、同 tweet id、相似标题/摘要指纹。
  - 聚类：同一事件多来源合并，保留“主来源 + 佐证来源”。
QualityScoringAgent：对每条内容打分并给出可解释明细（分项得分 + 命中规则 + 触发风险标签）。
Trend&InsightAgent：时间窗内热点主题、增长项、争议点、影响评估（对业务/市场/技术/监管）。
BriefComposerAgent：按简报模板产出（分栏、排序、引用、结论），并生成可供复核的“证据列表”。
FeedbackLoopAgent（可选，异步）：收集反馈、生成训练样本/规则调整、更新重排模型或阈值。

2. 数据采集方案（Twitter / YouTube / 公众号）

2.1 Twitter（X）怎么采集

关键词搜索，tweet 正文、引用/转推关系、回复线程（用于捕捉一手信息与争议）、媒体链接。
点赞、转推、回复、浏览（可得则取）、作者粉丝数、账号认证/创建时间。
优先捕捉“原始信息源”（原帖/官方声明/论文/产品发布页。

2.2 YouTube 怎么采集

采集入口：关键词搜索、频道订阅列表、播放列表、行业会议/官方频道。
采集内容：视频标题/简介/发布时间、字幕（有则抓；无则语音转写）、置顶评论与高赞评论。
采集指标：观看量、点赞、评论数、频道订阅数、视频时长、发布频率。

2.3 公众号怎么采集

采集入口：
可控方式：RSS/内容聚合接口、自建订阅源、第三方数据服务（合规前提）。
采集内容：标题、作者/公众号主体、正文、发布时间、原文链接、引用链接。。

### 2.4 技术限制与应对方案

限制：平台反爬/频控/账号风控：
应对：请求节流、指数退避、失败重试、按优先级抓取；必要时使用官方/合规数据服务。
限制：数据字段不一致、指标缺失：
应对：标准化数据模型（统一核心字段），缺失字段用“替代特征”补齐；在评分中对缺失做折扣而非直接淘汰。
限制：多语言/长内容成本高：
应对：先轻量摘要+关键词，再对高分内容做深度分析；分层处理降低 token 成本。
限制：事实准确性（谣言/截图流）：

3. “高质量”定义、冷启动过滤规则、反馈迭代机制（优势与亮点）

3.1 高质量数据的指标（可量化）

综合评分建议拆成可解释分项，便于调参：

相关性（Relevance）：与行业/主题词匹配度、实体命中（公司/产品/技术/政策）、语义相似度。
新颖度（Novelty）：是否为“最近更新内容”，是否近 7 天内容的最新内容。
可信度（Credibility）：来源权威（官方/机构/论文/一手发布）、可追溯引用（链接到原文/报告/PR）、多源交叉验证。
影响力（Impact/Signal）：互动量、增长率（单位时间增量）、作者影响力（粉丝/历史爆款率）、跨平台被引用次数。
信息密度（Information Density）：是否包含关键事实（时间/地点/数字/版本/价格/政策条款）、是否可抽取结构化要点。
可读性（Readability）：标题党/情绪化/纯观点水文降权；有段落结构、列表、引用、时间线加权。

输出时保留：**总分 + 分项分 + 命中证据**，让业务方能理解“为什么入选/为什么没入选”。

3.2 冷启动阶段（无用户反馈）如何过滤：规则优先、稳健保守

冷启动目标是“少而准”，宁可漏掉边缘信息，也避免简报变成信息噪声。
选用最新的内容实现高质量的数据

3.3 有用户反馈后怎么迭代：从规则到学习的闭环

引入反馈后，重点不是“立刻训练大模型”，而是用可控的方式提升排序与过滤效果。
有用/无用、重复、可信/不可信、太水/信息密度高、希望更多此主题。。
保留上周版本作为对照，监控关键指标（打开率/收藏率/有效阅读时长/投诉率）。

3.4 本方案的优势与亮点（总结）
先粗筛再深度数据分析，把 LLM 成本集中在高分内容上。
显著降低重复信息噪声，让热点更接近“事件”而不是“帖子的数量”。
提高数据检索效率
